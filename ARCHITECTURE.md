# FietsRouten — Architecture & Services

This document explains every component that makes the app work: what it does, why it exists, and how the pieces connect.

---

## Overview

```
Android App
    │
    ├── Map tiles (background map)  ──► OpenFreeMap CDN  (free, public)
    ├── Cycling routes / knooppunten overlay ──► Overpass API  (free, public)
    │
    └── Self-hosted backend (Docker on your VPS)
            ├── Nominatim      → address search / geocoding
            ├── GraphHopper    → route calculation
            ├── Tileserver-GL  → (previously used; now replaced by OpenFreeMap)
            └── POI Service    → restaurants, cafés etc. along a route
```

The self-hosted services run as Docker containers defined in `docker-compose.yml`. All four share a Docker network called `shared_proxy` (managed externally, e.g. by Nginx Proxy Manager) which handles HTTPS and domain routing to each service.

## Docker Services

### 1. Nominatim (`fietsnav-geocoding.learndelingo.nl`)

**What it does:** Converts text addresses into GPS coordinates (geocoding) and vice versa. When a user types "Stationsplein Utrecht" in the From/To fields, this service returns the latitude and longitude.

**Image:** `mediagis/nominatim:4.3`

**Key config:**
- Loads the Netherlands OSM extract (`netherlands-latest.osm.pbf`) into a PostgreSQL database on first startup — this takes ~30 minutes and only happens once.
- `REPLICATION_URL` keeps the data up-to-date by pulling daily diffs from Geofabrik.
- `shm_size: 1gb` — PostgreSQL needs extra shared memory for the initial import.
- Port `7070` on the host, `8080` inside the container.

**Used by the app:** `Config.NOMINATIM_BASE_URL` — the `actvFrom`, `actvTo`, and `actvPlannerCity` search fields query this for address suggestions.

**Data dependency:** Requires `data/netherlands-latest.osm.pbf` to exist before first startup (produced by `scripts/download-data.sh`).

---

### 2. GraphHopper (`fietsnav-routing.learndelingo.nl`)

**What it does:** Calculates cycling routes between two GPS coordinates. Given a start and end point, it returns a list of waypoints, turn-by-turn instructions, total distance, and estimated travel time.

**Image:** `israelhikingmap/graphhopper:latest`

**Key config (`graphhopper/config.yml`):**
- Reads the same Netherlands OSM PBF file to build an internal routing graph.
- Three cycling profiles are pre-computed:
  - **bike** — regular road cycling
  - **mtb** — mountain bike (prefers off-road paths)
  - **racingbike** — road bike (prefers smooth tarmac)
- `profiles_ch` — Contraction Hierarchies are enabled for each profile. This pre-processes the graph to make route queries near-instant (at the cost of a longer startup time when the graph is first built).
- `JAVA_OPTS: -Xmx2g -Xms512m` — GraphHopper is memory-intensive. It needs at least 2 GB of heap for the Netherlands graph.
- Port `8989`.

**Used by the app:** `Config.GRAPHHOPPER_BASE_URL` — called when the user taps "Bereken route" (address mode) or "Bereken route" (knooppunten mode). Also called automatically when the navigation engine detects the user has gone off-route.

**Data dependency:** Requires `data/netherlands-latest.osm.pbf`. On first start it builds a routing graph and caches it in the `graphhopper-cache` Docker volume (takes 5–15 min). Subsequent starts reuse the cache and are fast.

---

### 3. Tileserver-GL (`fietsnav-tile.learndelingo.nl`)

**What it does:** Serves vector map tiles and a map style from a pre-generated `.mbtiles` file. A vector tile contains raw geographic shapes (roads, buildings, water, etc.) which the app's MapLibre renderer draws on-screen.

**Image:** `maptiler/tileserver-gl:latest`

**Key config (`tileserver/config.json`):**
- Serves one style: `basic-preview`, loaded from `tileserver/styles/basic-preview/style.json`.
- Serves tile data from `data/netherlands.mbtiles`.
- Fonts (glyph PBF files for text labels) are mounted from `tileserver/fonts/`.
- Port `8080`.

**Current status — no longer used by the app.**
The app's base map style was switched to the free public OpenFreeMap CDN (`Config.MAP_STYLE_URL`). The Tileserver-GL container is still running in Docker and can be kept as a fallback or for offline use, but the Android app no longer calls it.

**Data dependency:** Requires `data/netherlands.mbtiles`, generated by `scripts/download-data.sh` using Tilemaker from the OSM PBF file.

---

### 4. POI Service (`fietsnav-pois.learndelingo.nl`)

**What it does:** A lightweight REST API that returns points of interest (restaurants, cafés, bars, etc.) within a bounding box. The app uses this to show food/drink stops along a planned route.

**Built from:** `pois-service/` — a custom Spring Boot 3 / Java 17 application, containerised via a multi-stage Dockerfile (Maven build stage → minimal JRE runtime stage).

**Stack:** Spring Boot Web + Spring JDBC + SQLite

**API endpoint:**
```
GET /api/pois?south=52.1&west=4.8&north=52.4&east=5.1&types=cafe,restaurant
→ GeoJSON FeatureCollection
```

Supported amenity types: `restaurant`, `cafe`, `fast_food`, `bar`, `bakery`, `ice_cream`, `pub`.

**Database:** Reads from `data/pois.db` — a SQLite file with a `pois` table indexed on `(lat, lon)` and `amenity`. SQL injection is prevented by a server-side allowlist of amenity types.

**Port:** `8090`.

**Used by the app:** `Config.POI_BASE_URL` — called when the user taps the POI FAB after a route is active. Results are then filtered client-side to only show POIs within 500m of the route line (haversine distance).

**Data dependency:** Requires `data/pois.db`, generated by running `scripts/extract_pois.py` against the OSM PBF file.

## Docker Volumes

| Volume | Used by | Purpose |
|---|---|---|
| `nominatim-data` | Nominatim | PostgreSQL database directory — persists the geocoding database across container restarts |
| `graphhopper-cache` | GraphHopper | Pre-processed routing graph — avoids rebuilding on every restart |

## Shared Network

All containers join `shared_proxy`, an **externally managed** Docker network (defined outside this compose file). In practice this is typically created by a reverse proxy stack (e.g. Nginx Proxy Manager or Traefik) running on the same server. The reverse proxy handles:
- HTTPS termination (SSL certificates via Let's Encrypt)
- Routing `fietsnav-geocoding.learndelingo.nl` → Nominatim (port 8080)
- Routing `fietsnav-routing.learndelingo.nl` → GraphHopper (port 8989)
- Routing `fietsnav-tile.learndelingo.nl` → Tileserver-GL (port 8080)
- Routing `fietsnav-pois.learndelingo.nl` → POI Service (port 8090)

## External / Public Services (no hosting required)

### OpenFreeMap — Map Tiles & Style
`https://tiles.openfreemap.org/styles/bright`

Provides the base map style and vector tile data the Android app renders as the background map. Free, no API key required, global CDN. Currently used for the "Bright" style (bold, colorful, Waze-like).

Dark mode fallback: `https://basemaps.cartocdn.com/gl/dark-matter-gl-style/style.json` (Carto, also free).

### Overpass API — Cycling Routes & Knooppunten Overlay
`https://overpass-api.de/`

A public query API for OpenStreetMap data. The app queries it for:
- **Named cycling routes** (LF routes, regional cycling networks) visible in the current map viewport — requested when the user taps the layers FAB.
- These are rendered as coloured line overlays on top of the base map.

The knooppunten (junction node numbers) are **not** fetched from Overpass at runtime — they are extracted once from the OSM PBF file at build time by `scripts/extract_knoopunten.py` and bundled directly into the app as `assets/knoopunten.geojson`.

### Geofabrik — OSM Data Downloads
`https://download.geofabrik.de/europe/netherlands-latest.osm.pbf`

The source for the raw OpenStreetMap data file for the Netherlands (~900 MB). Downloaded once by `scripts/download-data.sh`. Nominatim also uses Geofabrik's update feed (`netherlands-updates/`) for daily map data replication.

## Data Pipeline (one-time setup)

```
Geofabrik
    │
    └─ download-data.sh
            │
            ├─ netherlands-latest.osm.pbf ──► GraphHopper (routing graph, built on first start)
            │                             ──► Nominatim   (geocoding DB, built on first start)
            │                             ──► Tilemaker   → netherlands.mbtiles → Tileserver-GL
            │
            ├─ extract_pois.py  → data/pois.db         → POI Service (SQLite)
            └─ extract_knoopunten.py → knoopunten.geojson → bundled into APK (app assets)
```

Run order for a fresh deployment:
1. `bash scripts/download-data.sh` — downloads PBF, generates MBTiles (~30 min)
2. `python3 scripts/extract_pois.py` — extracts POIs into SQLite
3. `python3 scripts/extract_knoopunten.py` — extracts junction nodes into GeoJSON
4. `docker compose up -d` — starts all four services
5. Wait for Nominatim and GraphHopper to finish their first-start imports (check `docker logs` — can take 10–30 min each)
