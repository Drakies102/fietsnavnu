# fietsnavnu

Bike navigation stack for the Netherlands using GraphHopper, Nominatim, and tileserver-gl.

## Prerequisites

- Docker + Docker Compose
- ~10 GB free disk space

## First-time setup

Run the data download script once before starting the stack. It downloads the Netherlands OSM extract (~900 MB) and generates vector tiles (~10–30 min):

```sh
cd scripts
bash download-data.sh
```

Then start all services:

```sh
docker compose up -d
```

## Testing the endpoints

### GraphHopper (routing) — port 8989

```sh
# Health check
curl http://localhost:8989/health

# Bike route: Amsterdam → Utrecht
curl "http://localhost:8989/route?point=52.3728,4.8936&point=52.0907,5.1214&profile=bike&locale=nl"

# Available profiles: bike, mtb, racingbike
```

### Nominatim (geocoding) — port 7070

Nominatim takes ~20–30 minutes to finish its database import on first start. Check progress with `docker compose logs -f nominatim`.

```sh
# Health check
curl "http://localhost:7070/search?q=amsterdam&format=json&limit=1"

# Forward geocode
curl "http://localhost:7070/search?q=utrecht&format=json&limit=1"

# Reverse geocode (Dam Square, Amsterdam)
curl "http://localhost:7070/reverse?lat=52.3731&lon=4.8936&format=json"
```

### Tileserver (map tiles) — port 8080

Requires `data/netherlands.mbtiles` to exist (generated by `download-data.sh`).

```sh
# Health check
curl http://localhost:8080/health

# Tile metadata
curl http://localhost:8080/data/netherlands.json

# Fetch a vector tile (zoom 7, covers the Netherlands)
curl -o tile.pbf http://localhost:8080/data/netherlands/7/66/42.pbf
```

## Checking service status

```sh
# All containers
docker compose ps

# Live logs for a specific service
docker compose logs -f graphhopper
docker compose logs -f nominatim
docker compose logs -f tileserver
```
